# 甘南 + 莲宝叶则复核日程生成

更新日期：2026-06-25  
对应 SOP：[甘南莲宝叶则_反复核验SOP.md](甘南莲宝叶则_反复核验SOP.md)  
复核日志模板：[data/gannan_reverification_log.template.csv](data/gannan_reverification_log.template.csv)  
复核日历导出：[甘南莲宝叶则_复核日历导出.md](甘南莲宝叶则_复核日历导出.md)

这份文件把“反复验证”变成按日期执行的日程。只要确定兰州出发走甘南主线的 D1 日期，就能生成 T-14、T-7、T-3、T-1 和 D-day 每天早上的复核任务。

## 先看结论

生成 Markdown 给人看：

```bash
python3 scripts/gannan_reverification_schedule.py --route-start-date 2026-09-12
```

生成 CSV 作为真实复核日志草稿：

```bash
python3 scripts/gannan_reverification_schedule.py \
  --route-start-date 2026-09-12 \
  --format csv \
  --output data/gannan_reverification_log.csv
```

然后检查日志格式：

```bash
python3 scripts/check_gannan_reverification_log.py data/gannan_reverification_log.csv
```

生成手机/电脑日历提醒：

```bash
python3 scripts/gannan_reverification_calendar.py --route-start-date YYYY-MM-DD
```

## 日期口径

`--route-start-date` 指兰州正式出发去甘南的 D1 日期。  

- 如果前一晚已经到兰州，D1 就是“兰州 -> 夏河”的日期。
- 如果当天从外地飞到兰州并立刻出发，D1 仍然是兰州出发日期，但要额外确认航班不压缩 D1。
- 如果先安排 D0 兰州适应/取车，D1 应填第二天。

## 生成内容

| 轮次 | 生成内容 | 用途 |
|---|---|---|
| T-14 | 大交通、车辆、核心酒店、门票初查、预算 | 建立候选池 |
| T-7 | 交通短名单、车辆定稿、酒店电话确认、核心景区预约 | 第一次收敛 |
| T-3 | 天气、道路、莲宝叶则开放、机位光线、无人机、高原健康 | 风险复核 |
| T-1 | 次日路线、门票开放、主机位、酒店、身体和装备 | 执行复核 |
| D-day | 每天路线、门票、机位、身体状态 | 现场调整 |

## CSV 怎么用

生成的 CSV 与复核日志模板同列：

```text
round,check_date,category,item,source,source_url_or_contact,quoted_price_cny,evidence_file,status,decision,next_action,owner,next_check_date,notes
```

默认状态是 `n/a`，意思是“这是一条待执行任务，还不是复核结论”。实际查完后，把每行改成：

- `pass`：本轮证据足够。
- `watch`：可执行但有波动。
- `fail`：淘汰或改线。
- `blocked`：缺用户输入或外部回复。

同时补齐：

- `quoted_price_cny`
- `evidence_file`
- `decision`
- `next_action`
- `next_check_date`
- `notes`

## 和其他工具的关系

| 工具 | 作用 |
|---|---|
| `scripts/gannan_reverification_schedule.py` | 按日期生成要查什么 |
| `scripts/gannan_reverification_calendar.py` | 把复核任务导出为可导入日历的 ICS |
| `scripts/check_gannan_reverification_log.py` | 检查日志字段和状态是否合法 |
| `scripts/check_gannan_evidence.py --strict` | 检查日志里的证据截图是否真实存在 |
| `scripts/gannan_order_report.py` | 把候选、日志和红旗汇总成可读报告 |
| `scripts/run_gannan_checks.py --actual` | 真实订单阶段一键总核验 |
