# 甘南 + 莲宝叶则来源台账

更新日期：2026-06-25  
对应核验记录：[甘南莲宝叶则_核验记录.md](甘南莲宝叶则_核验记录.md)  
来源 CSV：[data/gannan_source_ledger.csv](data/gannan_source_ledger.csv)  
来源健康报告：[甘南莲宝叶则_来源健康报告.md](甘南莲宝叶则_来源健康报告.md)  
来源故障处理：[甘南莲宝叶则_来源故障处理.md](甘南莲宝叶则_来源故障处理.md)

这份台账把机票、酒店、门票、机位、天气、无人机和路线复核入口集中管理。它不等于“已下单价格”，只说明每轮复核应该去哪里查、查什么、用什么备选来源交叉验证。

## 使用方式

```bash
python3 scripts/check_gannan_sources.py
python3 scripts/check_gannan_sources.py --online
python3 scripts/gannan_source_health_report.py
python3 scripts/gannan_source_health_report.py --online
python3 scripts/gannan_source_fallbacks.py
python3 scripts/gannan_source_fallbacks.py --online
```

`check_gannan_sources.py` 默认只检查 CSV 格式、分类、优先级和 URL；加 `--online` 后会尝试访问外部链接。`gannan_source_health_report.py` 会生成来源健康报告；`gannan_source_fallbacks.py` 会生成来源故障处理清单。加 `--online` 后会把联网探测结果写入报告。部分 OTA、地图和政务站可能反爬或拒绝 HEAD 请求，脚本会把 `403 / 405 / 429` 作为“服务器可达但需要人工打开”的提醒，不把它当成价格或开放信息已核验。

## 分类口径

| 分类 | 用途 | 下单前怎么用 |
|---|---|---|
| flight | 兰州机场、航司、12306 等大交通入口 | 只认实际日期下单页最终含税价、行李和退改 |
| hotel | 酒店 OTA 和区域住宿入口 | 只认实际日期房态、房型、停车、热水、供暖和取消规则 |
| ticket | 景区门票、开放、观光车和临时公告 | T-7、T-3、T-1、当天早上反复查，不一次性买死 |
| route | 地图和路线关系 | 用于路线关系和大致耗时，现场以高德/百度、司机、道路公告为准 |
| photo | 日出日落、机位窗口 | 只给天文参考，实际见光要看山体遮挡和云量 |
| weather | 天气和预警 | 看雷雨、大风、降雪、道路风险，决定砍点 |
| health | 高原健康和高反应对原则 | 只做旅行健康红线参考，个人用药和慢病必须咨询医生 |
| drone | 无人机合规入口 | 实名和空域只是底线，现场禁飞牌优先 |
| budget | 成团产品和市场锚点 | 只做报价区间参考，不替代摄影包车成交价 |

## 必查优先级

| 优先级 | 含义 |
|---|---|
| A | 每轮关键复核入口，尤其门票、开放、天气、机场、无人机 |
| B | 交叉验证或辅助判断 |
| C | 价格锚点或补充信息 |

## 订单级使用边界

- 入口可打开，只能说明“来源仍可访问”，不能说明“价格未变”。
- OTA 票价和酒店价格必须回到实际日期下单页确认。
- 景区开放范围必须看官方公告、酒店前台、司机当地消息和当天现场牌示。
- 地图路线只提供规划参考，甘南/阿坝山路受天气、施工、牲畜和假期拥堵影响很大。
- 每次复核结果要写入 [data/gannan_reverification_log.template.csv](data/gannan_reverification_log.template.csv) 复制出的工作日志。

## 台账维护

新增来源时，先把它写入 CSV，再运行：

```bash
python3 scripts/check_gannan_sources.py
python3 scripts/verify_gannan.py
```

如果来源失效，不要直接删除；先在 CSV 的 `fallback` 和 `notes` 里写替代入口，再决定是否换源。
