# 甘南 + 莲宝叶则复核日历导出

更新日期：2026-06-25  
复核日程说明：[甘南莲宝叶则_复核日程生成.md](甘南莲宝叶则_复核日程生成.md)  
复核日志模板：[data/gannan_reverification_log.template.csv](data/gannan_reverification_log.template.csv)

这份工具把 T-14、T-7、T-3、T-1 和 D-day 复核任务变成可导入手机/电脑日历的 `.ics` 文件。导入后仍要把实际结果写回 `data/gannan_reverification_log.csv`，日历只负责提醒，不代表复核已经完成。

## 先看结论

没有真实 D1 日期时，不生成正式 ICS，避免把示例日期误当成下单计划。确定兰州出发走甘南主线的 D1 日期后运行：

```bash
python3 scripts/gannan_reverification_calendar.py --route-start-date YYYY-MM-DD
```

生成后会得到：

- `data/gannan_reverification_calendar.ics`
- 56 个日历事件
- T-14/T-7/T-3/T-1 晚上 20:00 复核提醒
- D-day 每天早上 07:00 现场复核提醒

## 导出命令

```bash
python3 scripts/gannan_reverification_calendar.py --route-start-date YYYY-MM-DD
python3 scripts/gannan_reverification_calendar.py --route-start-date YYYY-MM-DD --ics-output data/gannan_reverification_calendar.ics
python3 scripts/gannan_reverification_calendar.py --check
```

如果已经填好 `data/gannan_order_inputs.json` 里的 `trip.departure_date`，也可以直接运行：

```bash
python3 scripts/gannan_reverification_calendar.py
```

## 导入后怎么用

1. 把 `data/gannan_reverification_calendar.ics` 导入手机或电脑日历。
2. 每个提醒触发后，打开来源台账、来源故障处理和候选表做人工复核。
3. 复核结果写回 `data/gannan_reverification_log.csv`，证据截图放入 `evidence/gannan/`。
4. 下单前运行 `python3 scripts/gannan_booking_gate.py --strict`。
