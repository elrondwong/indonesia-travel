# 甘南 + 莲宝叶则证据归档规范

更新日期：2026-06-25  
对应订单复核报告：[甘南莲宝叶则_订单复核报告.md](甘南莲宝叶则_订单复核报告.md)  
证据目录：[evidence/gannan/](evidence/gannan/)

这份文件解决“反复验证以后证据放哪里”的问题。机票、酒店、包车、门票、机位、天气和景区开放不能只靠记忆或聊天印象；每轮复核都要留下截图、文字确认或现场照片，并在 CSV 的 `evidence_file` 里指向它。

## 先看结论

| 项目 | 必须留什么证据 | 写入哪个表 |
|---|---|---|
| 机票 / 高铁 | 最终含税价、行李、退改规则、起降时间截图 | `data/gannan_flight_candidates.csv` |
| 酒店 / 民宿 | 下单页、取消规则、真实房间/窗景、停车、热水/供暖、早出门确认 | `data/gannan_hotel_candidates.csv` |
| 包车 / 租车 | 报价、车型、座位/行李、费用包含项、保险、夜路政策、取消规则 | `data/gannan_transport_quotes.csv` |
| 门票 / 景区开放 | 票价、实名预约、观光车/扶梯、最早入园、最晚离园、开放范围 | `data/gannan_ticket_candidates.csv` |
| 机位 / 路况 | 高德/百度路线截图、司机定位、景区现场牌示、禁止进入/禁飞提示 | `data/gannan_reverification_log.csv` |
| 天气 / 高反 / 应急 | T-3/T-1/当天早上天气截图、改线原因、酒店/司机确认 | `data/gannan_reverification_log.csv` |

## 目录和命名

所有订单证据默认放在：

```text
evidence/gannan/
```

建议命名：

```text
YYYYMMDD_T-14_flight_outbound_option-a_price-policy.png
YYYYMMDD_T-7_hotel_zhagana_option-b_parking-hotwater.jpg
YYYYMMDD_T-3_ticket_lianbao_open-scope.png
YYYYMMDD_T-1_route_zhagana_lianbao_weather-road.png
YYYYMMDD_D-day_ticket_zhagana_gate-notice.jpg
```

命名口径：

- `YYYYMMDD`：复核当天日期。
- `T-14/T-7/T-3/T-1/D-day`：复核轮次。
- `flight/hotel/transport/ticket/route/photo/weather/drone/insurance`：证据类别。
- `option-a`：对应候选表里的 `option_id`。
- 最后一段写证据内容，例如 `price-policy`、`parking-hotwater`、`open-scope`。

## CSV 怎么填

在候选表或复核日志的 `evidence_file` 字段里写证据文件名：

```text
20260901_T-14_flight_outbound_option-a_price-policy.png
```

脚本会默认到 `evidence/gannan/` 查找。

也可以写完整相对路径：

```text
evidence/gannan/20260901_T-14_flight_outbound_option-a_price-policy.png
```

一个项目有多张截图时，用 `|` 分隔：

```text
hotel_price.png|hotel_cancel_policy.png|hotel_parking_chat.png
```

## 严格检查规则

默认命令只做温和检查，适合还没进入真实订单阶段：

```bash
python3 scripts/check_gannan_evidence.py
```

真实候选表和复核日志填好后，使用严格模式：

```bash
python3 scripts/check_gannan_evidence.py --strict
```

严格模式下会阻塞：

- 活跃候选没有 `evidence_file`。
- `evidence_file` 指向的本地文件不存在。
- 复核日志中 `pass/watch/fail/blocked` 状态没有证据。
- 证据路径逃出当前仓库。

一键真实订单核验会自动调用严格证据检查：

```bash
python3 scripts/run_gannan_checks.py --actual
```

## 不同证据的最低标准

| 类别 | 最低证据标准 | 不够用的证据 |
|---|---|---|
| 机票/高铁 | 下单页最终含税价 + 行李 + 退改规则同屏或分屏截图 | 搜索列表低价、只写航班号 |
| 酒店 | OTA 下单页 + 取消规则 + 电话/聊天确认停车热水早出门 | 平台宣传图、只看评分 |
| 包车/租车 | 文字报价 + 费用包含项 + 保险 + 夜路政策 | 只说“全包”但不列明 |
| 门票 | 官方/OTA 可预约页 + 开放范围 + 观光车/扶梯/最晚离园 | 游记价格、旧截图 |
| 机位 | 地图定位 + 现场开放/禁入/禁飞确认 | 只存网红图 |
| 天气路况 | T-3/T-1/当天截图 + 改线决策记录 | 只看一眼天气 App |

## 下单前证据清单

- [ ] 往返兰州大交通：价格、行李、退改、到达后交通。
- [ ] D0/D9 酒店或返程交通：不压缩主行程。
- [ ] 夏河酒店：停车、热水/供暖、早出门。
- [ ] 扎尕那酒店：真实窗景、停车、热水/供暖、早餐、到主机位距离。
- [ ] 阿坝县酒店：停车、热水/供暖、供氧或制氧机、能否帮问莲宝叶则开放。
- [ ] 包车/租车：车型、座位、行李、保险、费用包含项、夜路政策。
- [ ] 扎尕那：门票/预约/观光车/村内交通/洛克之路支线口径。
- [ ] 莲宝叶则：开放范围、景区交通、最早入园、最晚离园、天气关闭条件。
- [ ] 花湖和黄河九曲：观光车、扶梯、日落后下撤时间。
- [ ] 无人机：实名、空域、现场禁飞牌示。
- [ ] 天气和路况：T-3/T-1/当天早上截图。
