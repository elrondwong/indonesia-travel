这份 SOP 专门解决“反复验证”这件事：同一个机票、酒店、车辆、门票、机位和路况，不是查一次就结束，而是在 T-14、T-7、T-3、T-1 和当天早上按同一口径复核，留下证据，并根据触发条件决定下单、保留、降级或改线。

## 复核原则

1. 只认可追溯证据：下单页截图、官方/OTA 链接、酒店或司机文字回复、景区公告、电话记录摘要。
2. 价格只认最终含税价，不认搜索列表最低价。
3. 酒店只认明确房型、日期、取消规则、停车、热水和供暖回复。
4. 车辆只认写清车型、费用包含项、司机食宿、夜路态度和取消规则的报价。
5. 景区开放和票价只作为当轮结论，下一轮必须重查。
6. 任一核心项变红，不硬撑原计划；先保扎尕那全天、莲宝叶则全天和安全白天行车。

## 复核日志

复制模板后开始记录：

```bash
cp data/gannan_reverification_log.template.csv data/gannan_reverification_log.csv
python3 scripts/check_gannan_reverification_log.py data/gannan_reverification_log.csv
```

如果已经确定兰州出发日期，可以直接生成待复核日志草稿：

```bash
python3 scripts/gannan_reverification_schedule.py --route-start-date 2026-09-12 --format csv --output data/gannan_reverification_log.csv
```

每轮查完后追踪价格、状态和开放变化：

```bash
python3 scripts/gannan_reverification_diff.py --strict
```

每条记录至少填：

| 字段 | 怎么填 |
|---|---|
| round | T-14 / T-7 / T-3 / T-1 / D-day |
| check_date | 实际复核日期，格式 YYYY-MM-DD |
| category | flight / hotel / car / ticket / route / photo / drone / health |
| item | 具体项目，例如“扎尕那 D3-D4 民宿 A” |
| source | 航司 App、12306、携程、酒店微信、司机微信、景区电话等 |
| source_url_or_contact | 链接、电话、微信名或联系人 |
| quoted_price_cny | 最终人民币价格，没有价格就留空 |
| evidence_file | 截图或聊天记录文件名 |
| status | pass / watch / fail / blocked / n/a |
| decision | 下单、保留、继续观察、淘汰、改线 |
| next_action | 下一步动作 |
| owner | 谁负责跟进 |
| next_check_date | 下一次复核日期 |

## 复核时间轴

| 时间 | 核心目标 | 必须完成 |
|---|---|---|
| T-14 | 建立候选池 | 大交通 2-3 个候选；包车/租车至少 3 个报价；扎尕那和阿坝县可取消住宿先锁；核心门票可预约状态初查 |
| T-7 | 第一次收敛 | 选定交通方式；确认酒店真实房型、停车、热水、供暖、早出门；查景区预约、门票、开放和观光车 |
| T-3 | 风险复核 | 查天气、道路、景区临时公告；确认莲宝叶则开放范围；确认黄河九曲日落窗口；准备砍点顺序 |
| T-1 | 执行复核 | 确认次日出发时间、油量、早餐、门票、观光车、停车点、日出日落和替代机位 |
| D-day | 现场复核 | 看景区公告、司机判断、身体状态和天气变化；必要时立刻降强度或改线 |

## 分项复核清单

### 大交通

| 查什么 | 证据 | 变更条件 |
|---|---|---|
| 航班/高铁最终含税价 | 航司 App、12306、OTA 下单页截图 | 比同日相近时段高 30% 以上，继续比价或换时间 |
| 行李和摄影器材 | 票规截图 | 无托运或超重成本高，降级 |
| 到达兰州时间 | 订单页 | 晚于 18:00 到兰州，D0 只住兰州，不再长途 |
| 返程时间 | 订单页 | D9 早班机，D8 必须住兰州或机场附近 |

### 包车 / 自驾租车

| 查什么 | 证据 | 变更条件 |
|---|---|---|
| 车型和座位 | 司机/门店文字回复 | 行李装不下或低底盘车，淘汰 |
| 费用包含项 | 报价截图 | 不写油费、高速、停车、司机食宿，继续追问 |
| 夜路态度 | 司机文字回复 | 主动建议赶夜路，降级 |
| 路线经验 | 路线确认回复 | 未走过扎尕那/阿坝县/莲宝叶则且不愿核验，淘汰 |
| 保险/救援 | 租车平台或合同截图 | 轮胎、玻璃、救援不清楚，降级 |

### 酒店 / 民宿

| 区域 | 必须确认 | 变更条件 |
|---|---|---|
| 夏河 | 拉卜楞寺距离、停车、热水、供暖、早出门 | 早出门不方便或停车不明确，降级 |
| 合作 | 停车、早餐、电梯、补给 | 第二天早出发会被卡，降级 |
| 扎尕那 | 真实房型、窗景、停车、热水、供暖、晚餐、清晨出门 | 不发实拍或热水供暖含糊，淘汰 |
| 唐克/若尔盖 | 黄河日落后回酒店路况、晚到入住、晚饭 | 日落后回程太远，换近景区住处 |
| 阿坝县 | 供暖、热水、停车、供氧、能否帮问莲宝叶则 | 无法协助确认景区且基础设施差，降级 |
| 合作/临夏返程 | 到兰州机场/高铁站耗时、早退房、停车 | D9 交通压力大，改住兰州 |

### 门票 / 开放

| 景区 | T-7 | T-3 | T-1 / 当天 |
|---|---|---|---|
| 拉卜楞寺 | 查门票、开放和拍摄限制 | 查是否有法会或特殊管理 | 现场按寺院要求拍摄 |
| 甘加秘境 | 查票务组合和道路 | 查天气和道路 | 雨雾大则缩短 |
| 扎尕那 | 查门票、预约、观光车和村内交通 | 查限流、施工、天气 | 只选一个日落机位，不夜间乱换点 |
| 花湖 | 查门票/观光车 | 查栈道和天气 | 风雨大则缩短 |
| 黄河九曲 | 查门票/扶梯组合 | 查日落云量和闭园 | 保证日落后安全下撤 |
| 莲宝叶则 | 查门票和可预约日期 | 查开放范围、观光车/自驾规则 | 当天早上再问酒店/司机/景区，不开放就改阿坝县周边或返程 |

### 机位 / 拍大片窗口

| 机位 | 成功条件 | 失败替代 |
|---|---|---|
| 扎尕那日出 | 低云、晨雾、村寨层次、能安全到观景点 | 村内生活、人文细节、雨雾山体 |
| 扎尕那日落 | 天黑前到达固定机位 | 只拍村寨蓝调，不再赶第二机位 |
| 黄河九曲日落 | 17:00 前到唐克区域，云层有开口 | 拍河曲层次和栈道剪影，不等到闭园边缘 |
| 莲宝叶则倒影 | 风小、湖面平、区域开放 | 长焦压石山、低机位拍湖岸纹理 |
| 莲宝叶则蓝调 | 身体状态好、闭园时间允许 | 提前撤回阿坝县，保安全和第二天返程 |

### 天气 / 路况 / 高反

| 风险 | 证据 | 动作 |
|---|---|---|
| 连续降雨或雷暴 | 天气 App、中央气象台、司机当地消息 | 山路减速，砍沟谷和高位观景台 |
| 道路施工或封路 | 导航、司机、酒店前台 | 提前改住宿点或走分段返程 |
| 莲宝叶则大风、雷雨、降雪 | 景区/酒店/司机确认 | 不冲高位点，不硬等蓝调 |
| 高反症状 | 本人状态 | 停拍、休息、下降海拔，必要时就医 |

## 状态定义

| 状态 | 含义 | 处理 |
|---|---|---|
| pass | 本轮证据足够，暂时通过 | 保留或下单，下一轮再查 |
| watch | 可以执行但有波动 | 保留备选，设置下一次复核 |
| fail | 明确不适合 | 淘汰或改线 |
| blocked | 需要用户输入或等待外部回复 | 写清缺什么和谁负责 |
| n/a | 本轮不适用 | 不作为风险 |

## 改线触发器

- [ ] D0 晚到兰州：D1 只去夏河，不加甘加。
- [ ] D3 下午 16:00 后仍未接近扎尕那：只保入住和一个近机位。
- [ ] D5 17:00 前到不了唐克：放弃黄河九曲日落，不夜间硬赶。
- [ ] D7 莲宝叶则关闭或只开放很小范围：改阿坝县周边轻拍，保 D8 返程。
- [ ] D8 天气/体力差：住合作或临夏，不直冲兰州。
- [ ] 任一人明显高反：停止追机位，降低海拔或就医。

## 每晚 10 分钟复盘

```text
1. 明天最重要的一张照片是什么？
2. 明天最晚几点必须离开酒店 / 景区 / 机位？
3. 哪个小点可以先砍？
4. 门票、停车、早餐、油量、充电、衣服是否已确认？
5. 今天照片是否已经双备份？
6. 明天如果天气失败，替代拍法是什么？
```

## 最终下单口径

当一个候选同时满足下面条件，才从“候选”变成“可下单”：

- [ ] 日期、人数、房型或车型完全匹配。
- [ ] 最终含税价已截图。
- [ ] 取消/退改规则已截图。
- [ ] 关键执行条件已确认：停车、早出门、热水、供暖、行李、夜路边界。
- [ ] 至少有一个备选方案。
- [ ] 已写入复核日志和预算工作台。
