## 不能提前写死的项目

| 项目 | 为什么不能提前写死 | 何时复核 | 复核口径 |
|---|---|---|---|
| 机票/高铁成交价 | 日期、出发城市、行李、退改规则决定价格 | T-14/T-7/T-3 | 航司 App/12306 + OTA，按最终含税价 |
| 酒店成交价 | 旺季房态、房型、是否可取消变化很大 | T-14/T-7 | OTA 下单页 + 酒店电话 |
| 包车/租车报价 | 车型、天数、司机食宿、油费、高速、停车差异大 | T-14/T-7 | 至少 3 家报价，写清包含项 |
| 扎尕那票价 | 公开平台有 80、80+40、110 等口径 | T-7/T-3/T-1 | 官方小程序/景区公告/酒店确认 |
| 莲宝叶则开放范围 | 高海拔景区受天气、道路、管理影响 | T-3/T-1/当天早上 | 景区公告、酒店前台、司机当地消息 |
| 花湖/黄河九曲组合票 | 门票、观光车、扶梯组合可能变化 | T-1 | 按是否追日落和体力选择 |
| 天气和路况 | 甘南/阿坝午后雷阵雨、塌方、施工强时效 | T-3/T-1/当天 | 中国天气、中央气象台、司机当地消息 |

- [ ] 机票/高铁不是只看最低价，已确认行李和退改
- [ ] D0/D9 不被航班时间压缩
- [ ] 包车/租车费用包含项写清楚
- [ ] 夏河、扎尕那、阿坝县酒店已电话确认热水、供暖、停车、早出门
- [ ] 扎尕那门票和观光车口径已二次确认
- [ ] 莲宝叶则开放范围、最早入园、最晚离园已二次确认
- [ ] 花湖/黄河九曲是否买组合票已按当天计划确认
- [ ] 保险已买
- [ ] 机动预算至少 10%
- [ ] 所有订单截图离线保存

## 本地页面完整性核验

每次修改甘南攻略包后，先重新生成 HTML，再运行核验：

```bash
python3 scripts/build_gannan.py
python3 scripts/verify_gannan.py
```

其中 `build_gannan.py` 会给所有 Pandoc 生成的甘南页面加统一顶部导航；`verify_gannan.py` 会检查文件、关键章节、站内链接和原印尼页面的“甘南攻略”入口。

单独运行核验命令：

```bash
python3 scripts/verify_gannan.py
```

通过时会输出：

```text
Gannan guide verification passed.
```

## 当前结论

这套攻略已经满足“可执行攻略包”的要求：有路线、有机位、有摄影执行、有装备、有沟通话术、有门票/酒店/机票核验框架、有预算工作台、有路线图和网页入口。  

它还不能替代“实际日期的订单核验”：没有出发日期、人数和预算时，写死具体机票和酒店价格反而会误导。下一步只需要补齐出行信息，就能把候选航班、候选酒店、包车报价和门票预约逐项填进预算工作台。
